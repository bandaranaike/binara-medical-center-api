# Task Workflow

Use this folder to track work clearly.

## Files

- `active/current-task.md`
  - one actively maintained task note for the work in progress
- `inbox.md`
  - user-facing drop point for new tasks, rough notes, and follow-ups
- `backlog/`
  - deeper task notes that are not currently active

## Rules

- Keep `active/current-task.md` focused on the one task being executed now.
- Move finished or paused work out of `active/` and into `backlog/` only if it still has future value.
- Treat `inbox.md` as raw input. Clean it up only when a task is actually started.
- Do not let domain reference notes live in `tasks/`; those belong in `domains/`.
